# 事件的补充

在Vue里面，我们的Vue的事件当中在传参的时候，不能随便使用this这个对象，如果要获取事件触发的对象，我们需要通过事件对象Event来获取元素

同时，Vue当中，传递事件的时候，我们使用$event来进行事件对象的传递，同时，如果你里面没有其它的参数，你可以把事件后面的方法的括号给省略掉，具体代码如下：

<body>
    <div id="app">
        <div class="div1" @click="test($event)">这是一个DIV</div>
        Vue的事件里面，可以不要括号，不要括号以后也可以调用方法，同时会根据你的参数自动的注入事件对象
        <div class="div1" @click="test2">这又是一个DIV</div>
    </div>
</body>
<script>
    new Vue({
        el: "#app",
        methods: {
            test(event) {
                //这个地方的event就是这里面的事件对象
                //这种方法在火狐里面不适用，报错，提示event是undefined
                //这一种传event的方式 在火狐时面也不适用
                //vue里面解决了这一个兼容性event ----- $event
                console.log(event.currentTarget);
                //要求：我希望点击以后，拿出里面的DIV内容
                console.log("");
            },
            test2(event) {
                console.log(event);
            }
        }
    })
</script>

# Vue中DOM对象的操作方法

在Vue里面，如果要拿到其它的DOM对象，首先，我们可以使用下面两种方法

  1. 传统的DOM操作document.getElementById(),document.querySelector().......

  2. 通过Vue特有的属性$refs来进行DOM的操作

    <body>
    <div id="app">
        <p ref="p1">这是一个P标签</p>
        <input type="text" ref="input1">
        <hr>
        <div class="div1" @click="test($event)">这是一个DIV</div>
        Vue的事件里面，可以不要括号，不要括号以后也可以调用方法，同时会根据你的参数自动的注入事件对象
        <div class="div1" @click="test2">这又是一个DIV</div>
        <button type="button" @click="test3">获取P标签的内容</button>
    </div>
</body>
<script>
    new Vue({
        el: "#app",
        data: {

        },
        methods: {
            test(event) {
                //这个地方的event就是这里面的事件对象
                //这种方法在火狐里面不适用，报错，提示event是undefined
                //这一种传event的方式 在火狐时面也不适用
                //vue里面解决了这一个兼容性event ----- $event
                console.log(event.currentTarget);
                //要求：我希望点击以后，拿出里面的DIV内容
                console.log("");
            },
            test2(event) {
                console.log(event);
            },
            test3() {
                // var p = document.querySelector("p");
                // console.log(p);
                //console.log(this); //这个this指的是Vue
                console.log(this.$refs.p1);
                console.log(this.$refs.input1);
            }
        }
    })
</script>

# Vue组件及组件化开发

在Vue当中，我们可以把某一些特定的重复使用的功能封装起来，做成一个组件，这个组件大家可以把它理解我们之前学习的 分布模板

Vue的高级部分以及单页面开发的模块化全部都是基于组件化进行的开发，现在我们来了解一下Vue当中的组件

在Vue当中，Vue的组件分为两种，一种是全局组件，一种是局部组件，它们的使用点了各不相同，具体请看下面操作

组件分为两个部分，第一个部分是我们的模板，第二个部分是我们的数据，我们在定义Vue组件的模板template的时候，有三种方式去定义：

# 组件使用的注意事项

  1. 组件里面的template不支持片段代码，也就是template的标签下面，只允许有一个根标签
  2. 组件里面的template不能写在Vue的托管范围之内
  3. Vue定义的组件名称（无论是全局组件还是局部组件），如果组件的名子采用了驼峰命名，那么在使用这个组件的时候，必须要经过转义，例如：你定义组件名称叫userInfo,那么，在使用这个组件的时候，我们就需要转换成<user-info></user-info>

# 组件中使用插槽slot

Vue的组件默认情况之下是不允许插入内容的，即使插入也不会显示，但有时候，我们在定义组件的时候，我们模板里面，有些地方确实需要插入一个内容，为了解决这个问题，我们在template里面，有一个标签叫<slot>,这个标签相当于一个占位符，在模板template里面，占一个位置，你可以在需要的时候向里面插入东西，不需要就算了

<template id="temp1">
    <div>
        <!--我现在想在这里插入一个标题，但这个标题不确定是几号标题，有可能是h1,也有可能是h6 -->
        <ul>
            <li>张三</li>
            <li>李四</li>
            <li>王五</li>
        </ul>
        <slot></slot>
    </div>
</template>

# 组件中的数据定义

在Vue的组件当中，我们还可以向里面定义一些数据，用这些数据去处理我们的组件

Vue组件当中的数据主要有两种：第一种就是通过自身去定义的数据；第二种就是通过外面传递进来的数据（父级组件传递）

  • 组件自身定义的数据

    在Vue里面，我们也可以通过data去接管这个组件内（template）区域的数据，但是它不像Vue里面。在Vue里面Data是一个对象，但组件里面，它的data是一个方法，这个方法返回的数据是一个对象，这个对象才是页面上面托管的数据

    <body>
    <div id="app">
        <user-info>
            <h2 slot="header">这是里面的内容</h2>
            <p slot="footer">这是下面的内容</p>
        </user-info>
    </div>
    <template id="temp1">
        <div>
            <!--我现在想在这里插入一个标题，但这个标题不确定是几号标题，有可能是h1,也有可能是h6 -->
            <slot name="header"></slot>
            <ul>
                <li v-for="(item,index) in userList">{{item}}</li>
            </ul>
            <slot name="footer"></slot>
        </div>
    </template>

</body>
<!-- 
  Vue提供组件的封装功能用以简化我们的HTML代码量
-->
<script>
    //在Vue当中，注册局步的组件只需要创建一个对象就可以了，这个对象里面，需要有一个template的属性
    var userInfo = {
        template: "#temp1",
        //一定要注意，组件里面的data是一个方法，返回的对象才是页面上面的数据
        data() {
            return {
                userList: ["张三", "李四", "王五", "赵六", "标哥哥"]
            }
        }
    };
    //使用组件  全局组件注册完成以后，可以直接使用
    new Vue({
        el: "#app",
        components: {
            // user: user
            userInfo
        },
        data: {

        }
    });
</script>

# 组件中数据流的单向性

在Vue里面，数据流的方向是通过父组件转向子组件，如果子组件改变以后，是不能够去改变父组件的值的，这就是Vue提到的数据流的单向性

<body>
    <div id="app">
        <p>父组件：{{userName}}</p>
        <temp1 :user-name="userName"></temp1>
    </div>
    <template id="temp1">
      <div>
          <p>子组件：{{userName}}</p>
          <button type="button" @click="changeValue">子组件的按钮改变userName</button>
      </div> 
    </template>
</body>
<script>
    //注册一个组件
    var temp1 = {
        template: "#temp1",
        props: ["userName"],
        methods: {
            changeValue() {
                this.userName = "张三";
            }
        }
    };
    new Vue({
        el: "#app",
        data: {
            userName: "杨标"
        },
        components: {
            temp1
        }
    })
</script>

如果数据流确实是单向性的，那么，我改变父组件的值以后，子组件应该会改变

# 破坏数据流的单向性

破坏数据流的单向性我们指的就是在子组里面去改变从父级组件接收过来的值，同时，也改变父组件的这个值。这一种方式在Vue里面，不推荐，但是有解决办法！

  1. 通过对象的原理去改变数据【基础知识自已得到的方法】

    对象在做传递的时候用的是地址传递（引用传递），这一种传递有它的优势，现在我们就利用这一个特点，改变它数据流

    <body>
    <div id="app">
        <p>父组件：{{obj.userName}}</p>
        <button type="button" @click="obj.userName='张三李四王五'">父组件按钮改变父组件userName的值</button>
        <hr>
        <temp1 :obj="obj"></temp1>
    </div>
    <template id="temp1">
      <div>
          <p>子组件：{{obj.userName}}</p>
          <button type="button" @click="changeValue">子组件的按钮改变userName</button>
      </div>
      
    </template>
</body>
<script>
    //注册一个组件
    var temp1 = {
        template: "#temp1",
        props: ["obj"],
        methods: {
            changeValue() {
                this.obj.userName = "张三,我在子组件里面改变的他的值";
            }
        }
    };
    new Vue({
        el: "#app",
        data: {
            obj: {
                userName: "杨标"
            }
        },
        components: {
            temp1
        }
    })
</script>

  2. 通过事件去改变【官方推荐的方法】

    在Vue的官方文档里面，如果要改变数据的单向性问题，必须通过自定义的事件来完成，而自定义事件的触发也是有一定的条件

    <body>
    <div id="app">
        <p>父组件：{{userName}}</p>
        <button type="button" @click="userName='张三李四王五'">父组件按钮改变父组件userName的值</button>
        <hr>
        <temp1 :user-name="userName" @aaa="abc"></temp1>
    </div>
    <template id="temp1">
      <div>
          <p>子组件：{{userName}}</p>
          <button type="button" @click="changeValue">子组件的按钮改变userName</button>
      </div>
      
    </template>
</body>
<script>
    //注册一个组件
    var temp1 = {
        template: "#temp1",
        props: ["userName"],
        methods: {
            changeValue() {
                //$emit发射 触发
                this.$emit("aaa", "我是改变以后的子组件的值");
                //$emit不仅可以触发自定义事件，也可以触发内置事件
            }
        }
    };
    new Vue({
        el: "#app",
        data: {
            userName: "杨标"
        },
        components: {
            temp1
        },
        methods: {
            abc(msg) {
                this.userName = msg;

            }
        }
    })
</script>

                <input :type="type==undefined?'text':type" @input="$emit('input',$event.currentTarget.value)" :value="value" class=" form-control" :placeholder="title">
            </div>
        </div>

    //注册全局组件
    Vue.component("inputItem", {
        template: "#inputItem",
        props: ["label", "title", "value", "type"]

        el: "#app",
        data: {
            formData: {
