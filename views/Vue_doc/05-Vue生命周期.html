# Vue生命周期及钩子函数

Vue的对象是我们在JS里面通过new产生出来的这一个对象，这个对象由创建到销毁有一个过程，我们以前使用vue,只知道new一个vue对象出来就行了，然后不管了，我们并没有去深究这个vue在创建以后，它是如何去运行的，如何去把页面上面的内容渲染出来的

vue的内部，有一些特定的方法，这些方法会随着vue的创建与销毁而自动去执行，这一种方法我们把经叫做钩子函数

# Vue的生命周期

Vue的八种状态分别也代表了它的八个钩子函数

在Vue里面，我们把Vue分解成四个过程，八种状态，这八种状态根据它由创建到销毁的过程，我们列出如下顺序

  1. beforeCreate，这个状态表示Vue在创建之前
    • 在这个状态下在，它不能调用data时面的数据
    • 可以使用$refs引用来操作DOM对象
    • 它不可以调用内部方法(methods里面定义的方法)
  2. created表示Vue已经创建完成
    • 它可以调用data当中的数据
    • 可以使用$refs引用来操作DOM对象
    • 可以调用methods里面定义的方法
  3. beforeMount，这个状态表示挂载之前
  4. mounted,这个状态表达挂载

    created:在模板渲染成html前调用，即通常初始化某些属性值，然后再渲染成视图。mounted:在模板渲染成html后调用，通常是初始化页面完成后，再对html的dom节点进行一些需要的操作

  5. beforeUpdate 组件更新以前
  6. updated 组件更新以后
  7. beforeDestory 销毁之前
  8. destoryed 销毁以后

在上面的两个状态里面，它都属于一个创建的过程，只在在创建的时候，一个是创建之前，一个是创建之后，在这两个状态里面，有些注意事件需要大家注意

在上八个状态里面，我们有些地方特别是在beforeCreated我们是无法获取到data这个对象的，但是，我们有一些特定的方法可以处理$nextTick这个方法可以帮我们完成这个选择过程

重要点：Vue当中的8个状态，4个过程适用于Vue的对象，也同样适用于Vue的组件，它们的生命周期过程是一样的，每一个组件都有一个生命周期

根据这个生命的特征，我们在v-if与v-show上面做了个允份的验证过程，我们发现，v-if把当前的这个组件条件不成立后就销毁掉了，条件成立以后又自动创建，但不停的去执行它的生命周期

而v-show则没有，它仅仅只用创建一次，然后条件不成立以后，就隐藏起来，并没有销毁！
