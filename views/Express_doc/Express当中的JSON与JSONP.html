# Express当中的JSON与JSONP

# JSON的前后端处理，以及JSON的Content-Type

JSON是一各理想化前后端数据交换格式，在Express当中，我们如何去使用JSON，并把JSON传递到前端呢

  1. 首先，我们选在Express的后台程序当中，取出自己要传递到前面的对象，然后通过JSON.stringify()这个方法把对象序列化成json字符串，然后通过resp.send()把转换好的json字符串发送到前台
  2. 前端接收到这个json字符串以后，在这个地方，可以通过JSON.parse() 这个方法把我们的json字符串转换为js对象

具体代码如下所示：

router.post("/studentListAjax",async (req,resp)=>{
  var studentInfoService=new StudentInfoService();
  try {
    var result=await studentInfoService.getAllList();
    //我现在希望把这个result转成json然后传递到前台去
    var jsonStr=JSON.stringify(result);
    resp.send(jsonStr);
  } catch (error) {
    resp.send("数据库错误");
  }
});

我们看到，在这里，我们拿到的这个data结果就是后台通过resp.send()发送过来的json字符串，然后我们在前台调用JSON.parse(data)这个方法把json字符串转换成了js对象

在此得我们就不得不思考一个问题，本来在后台就是一个对象，现在，将它转成字符串以后传递到前面，我们又需要将这个字符串转换成对象，这样就会造成很多时间浪费，与会降低效率，我们现在就需要去解决这一个问题

我们通过这这个Ajax的Network的监控请求发现，我们的请求响应信息Response Header这个返回的里面，有一个属性叫Content-Type:text/html;charset=utf8,这个是设置当前的响应信息的，我们现在在这个地方只需要去改变这个Content-Type的值为application/json;charset=utf8就可以了，关键问题是我们怎么去改呢？

Response Header代表的是服务器返回的头部信息，我们要改这个信息就需要返回到服务器！OK！，我们现在就回到服务器，去添加一个头部信息，代码如下：

router.post("/studentListAjax",async (req,resp)=>{
  var studentInfoService=new StudentInfoService();
  try {
    var result=await studentInfoService.getAllList();
    //我现在希望把这个result转成json然后传递到前台去
     var jsonStr=JSON.stringify(result);
     resp.setHeader("content-type","application/json");
      //上面的代码就是添加头部信息
     resp.send(jsonStr);
  } catch (error) {
    resp.send("数据库错误");
  }
});

在上面的代码当中，我们发现，resp.setHeader("content-type","application/json");这句代码的意思就是在我们当前的响应头部去添加了一个Content-Type,并且将它的值设置成了application/json;charset=utf8;这个时候，前台的页面接收到的这个结果就自动转换成了js对象了

可是我们不可能每一次要向前台传递数据都使用这个去设置，这样很麻烦

在Express里面，Express提供了一种简单的方法，可以直接完成上面的操作，这个方法就是resp.json(),在这里，我们贴上我们的代码

router.post("/studentListAjax",async (req,resp)=>{
  var studentInfoService=new StudentInfoService();
  try {
    var result=await studentInfoService.getAllList();
    //我现在希望把这个result转成json然后传递到前台去
    resp.json(result);
  } catch (error) {
    resp.send("数据库错误");
  }
});

# JSONP的后台实现

我们学过Ajax以后，我们知道了Ajax的一个很大的缺点就是不能够进行跨域，现在在这里，我们就通过jsonp一现实一次跨域的请求信息

jsonp的主要设置点还是在后台，我们要了解一点，jsonp返回的本质上面是一个方法，也就是后台要返回一个方法到前台当中去，这个时候，在这里，我们需要进行如下操作

  1. 第一步，先设置express的jsonp callBack的名称，设置如下：

    app.set("jsonp callback name","cb");

  2. 在程序当中，把需要的数据通过jsonp返回到前端

    router.get("/studentListJsonp",async (req,resp)=>{
  var studentInfoService=new StudentInfoService();
  try {
    var result=await studentInfoService.getAllList();
    resp.jsonp(result);
  } catch (error) {
    resp.send("数据库错误");

注意：jsonp本质上面并没有使用XMLHttpRequest这个对象来进行请求，而是使用的HTML标签<script></script>这个来进行请求，因为<script>标签的src属性有跨域的特点，所以，我们可以借助这个特别访问到跨域以后的内容。

# Ajax跨域的后台处理方式

在一些程序当中，我们会经常碰到一些问题，就是整个程序的我们都需要进行跨域，但是又不想进行jsonp跨域，因为jsonp是其于GET请求的，而GET请求是通过浏览器的地址栏在进行访问，在这里，GET请求的限制就体现出来了（例如文件上传，大量的数据提交等，这些都不能进行GET操作）

这个时候，我们就需要在此处使用一个特定的方法，让整个程序都可以进行Ajax的跨域访问

app.use((req,resp,next)=>{
    resp.setHeader("Access-Control-Allow-Origin","*");
    resp.setHeader("Access-Control-Allow-Methods","GET,POST,PUT,DELETE,OPTIONS");
    next();
})
