# Cookie与Session在Express项目中的使用

# Express中的Cookie使用

在express中，如果需要使用cookie,那么，我们就需在安装Express的Cookie中间件，在Express4里面，Express已经去掉了默认的Cookie功能 ，需要我们手动去安装

cnpm install cookie-parser --save
1

经过上面的安装以后，我们就可以使用cookie了，使用步骤如下

  1. 第一步先导入Cookie的中间件

    const cookieParser=require('cookie-parser');
    1

  2. 第二步，在当前程序当中使用中间件

    app.use(cookieParser("secret"));
    1

    在此处需要说明一点，我们在这个地方向里面传递了一个参数，这个参数是用于在服务器设置cookie的时候加密的时候使用的

  3. 第三步：服务端设置cookie值

    resp.cookie("name","value",{signed:true}); 
//这个地方后面的一个参数signed就是对cookie进行了加密
    1
    2

    **注意：**在此得需要注意，我们这里如果要使用加密的cookie,则在app.use(cookieParser(secret))的时候一定要向里面添加参数，否则这个地方会报错

  4. 第四步：Cookie取值

    cookie的取值分为两种情况，第一种情况就是没有加密的cookie

    req.cookies.name    //这个地方的name是属名
    1

    而对于加密的cookie，我们则需要使用另一种方式来取值

    req.signedCookies.name   //这个地方的name是属性名
    1

经过上面的四个步骤以后，我们其实已经把cookie的一些用户基本掌握了，但是在这个地方要说明一点，cookie是保存在浏览器里面的，而我们竟然可以在服务器端使用cookie值，这个地方就说明cookie会随着每一次的请求发送到服务端，这里，同学们可以通过浏览器里面的审查元素抓包，会看到request的请求头里面包含cookie的相关信息

既然cookie会随着我们的浏览器的请求发送到服务器这个时候，我们就需要对cookie的大小做一些限制，因为如果cookie可以无限增大，那么当用户访问一个服务器的时候，就会变得非常慢【因为这个时候的请求当中包含了cookie的信息】，所以我们的cookie有一些大小的限制，具体如下：

  1. cookie会在请求时发送到服务器，作为会话标识，服务器可修改cookie；
  2. 浏览器不能保存超过300个cookie，单个服务器不能超过20个，每个cookie不能超过4k

因为cookie可以在浏览器端通过document.cookie来获取，所以，浏览器可以修改我们服务器端创建的cookie，这个时候就很不安全，所以在这个地方，我们要慎用，不要将一些敏感的信息保存的cookie当中

# Express当中的Session的使用

Express看人session大家可以把它看成是一前端浏览器当中的sessionStorage这个对象，但它与这个对象又有一些不同的地方，也有一些相同的地方，现在，我们就在这里分析一下它们的相同点：

  • 相同点
    1. 它们都是以浏览器为单们，不能跨浏览器
    2. 它们默认都是关闭浏览器以后自动消失
  • 不同点
    1. session是存放在服务端的，而sessionStorage是存放在浏览器端
    2. session是在整个后台程序当中都可以使用的，而sessionStorage是默认不能跨页面存取值的（window.open()这个方法打开的父子页面是可以共享sessionStorage的值的）；

现在，我们就开始在后台的Express程序当中去使用session，同样，使用session我们也需要安装中间件，Express3的版本当中集成此对外，在Express4的版本当中，我们需要手动安装配置，具体步骤如下：

  1. 安装session中间件

    cnpm install express-session --save
    1

  2. 导入并加载session中间件

    const session=require('express-session');
app.use(session({
    secret:"abcdaaa", //secret:用来注册session id 到cookie中，相当与一个密钥。
  	resave:true,
  	saveUninitialized:true   //是否设置session在存储容器中可以给修改
}));
    1
    2
    3
    4
    5
    6

  3. 当加载上面的中间件以后，我们就可以在req这个对象当中使用我们的session了，具体如下

    var userInfo={
    s_id:'H17060001',
    s_name:'学生'
};
req.session.userInfo=userInfo;
    1
    2
    3
    4
    5

    经过上面的代码以后，我们就在这个地方把userInfo这个放在了session里面去了

    说明：在Express当中，cookie只能存放字符串，而session可以存放对象

# 通过session对程序做登陆验证

我们刚刚学习session的时候，我们都知道 ，session是一个全局的，只要有req对象的地方，我们就能够得到session，并且我们还知道，当一个浏览器断开与服务器的连接以后，它的session就会消失掉，我们可以通过这个特性对我们的程序做一个简单的登陆验证，具体如下：

app.use((req,resp,next)=>{
    if(req.session.stuInfo==undefined){
        //说明没有登陆
        //如果你访问的是登陆页面，这个时候，我就要让你放行
        if(req.url=="/StudentInfo/login"||req.url=="/StudentInfo/checkLogin"){
            next();
        }
        else{
            //重新跳转到登陆页面
            //resp.redirect("/StudentInfo/login");
            //next();
            resp.send("<script>top.location.href='/StudentInfo/login';</script>")
        }
    }
    else{
        //说明已经登陆过了
        next();
    }
});
