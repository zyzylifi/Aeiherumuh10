            # Express创建过程

            1. 初始化当前项目cnpm init --yes 初始化成功以后，这个时候会帮我们生成一文件，这个文件是什么package.json,就是当前项目的配置文件以后无论大家是在做前台项目还是后台项目，我们这个地方只要有node平台，都需要初始化

            2. 安装依赖项

              Express它是一个框架，并不是我们的nodejs自带的一个模块，这个时候，我们需要去使用它，下载它

              cnpm install express --save 后面的--save代表的就是把当前这条安装记录保存在我们的项目配置文件package.json，--save它的官方名称叫运行时依赖，【生产环境】依赖 ，它的简写方法就是-S;

            3. 按照最原始的开发思路，在这里， 我们需要导入这个模块，创建application,如果说这个Express它是3这个版本，那么我们这个程序app它需要运行在一个服务器，这个时候，我们需要使用http这个包来创建一个http服务

              const http=require('http');
const express=require('express');
const app=express();     //通过这个方法，它返回了一个app对象，这就是当前的程序

//Express3 创建必须如下
//https创建也需要使用下面的方法
//创建http服务，并且监听端口号
/*
http.createServer(app).listen(80,()=>{
    console.log("服务器创建成功");
})
*/
//在Express4的时候，我们就简化了我们的操作过程了
app.listen(80,()=>{
    console.log("服务器创建成功");
})

            5. 在Express时面，怎么样去使用模板

              • cnpm install ejs --save 去安装模板模块

              • 指定当前程序的模板信息

                app.set("views",path.join(__dirname,"./目录");       //设置模板文件的文件夹
app.set("view engine","ejs");                //指定模板引擎为ejs
                1
                2

                以上的模板设置是初始的模板设置，这个时候，我们的模板名称的后缀名为.ejs,如果想改变它的后缀名，我们需要进行另外的设置

              • 更改ejs模板的后缀名为html

                app.set("views",path.join(__dirname,"./目录");       //设置模板文件的文件夹
app.engine(".html",ejs.__express);			//注册一个.html后缀的文件为模板
app.set("view engine","html");

            # GET、POST、其它方式的取值

            1. get的取值方式req.query这个对象里面直接去拿属性

            2. post的取值是需要我们去加载一个中间件去取值body-parser以及multer这两个，它的取值使用req.body

            3. 路径变量

              • 写法如下：
              app.get("/login/:userName/:pwd",(req,resp)=>{
  var userName=req.params.userName;
  var pwd=req.params.pwd;
  console.log(userName,pwd);
});
              1
              2
              3
              4
              5
              • 取值方式:req.params.路径变量;
              • req.param() 这个方法也可以进行取值，这个方法既可以取get的值，也可以取post的值，还可以取路径变量的值
