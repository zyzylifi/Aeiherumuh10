    Zeno 2020-03-10 Express

    # art-template在Express中的使用

    art-template是一个前后端通用的模板，现在在这里， 我们将它应用在express的后面项目当中，具体步骤如下：

    1. 安装art-template模板

      cnpm install art-template express-art-template --save
      1

    2. 在express的项目当配置并使用art-template模板

      const template=require('express-art-template');
app.set("views",path.join(__dirname,"./views"));
app.engine(".html",template);
app.set("view engine","html");
      1
      2
      3
      4

    3. art-template在页面当中进行模板渲染

      <ul class=" list-group">
    {{each stuList item index}}
    <li class=" list-group-item {{index%2==0?'list-group-item-info':''}}" data-sid="{{item.s_id}}">{{item.s_name}}---{{item.s_sex}}</li>
    {{/each}}
</ul>
      1
      2
      3
      4
      5

      上面的模板当中，我们遍历了一个集合对象stuList，后面的两个值item代表的是当前遍历出来的对象，而index代表当前遍历的索引，如果在这个地方，我们不指定item与index,那么，可以通过$value与$index来获取

      上面的写法，是一个简化的写法，完整写法如下

      <ul class=" list-group">
    {{each stuList item index}}
    {{if(index%2==0)}}
    	<li class=" list-group-item">{{item.s_name}}---{{item.s_sex}}</li>
    {{else}}
    	<li class=" list-group-item list-group-item-info">{{item.s_name}}---{{item.s_sex}}</li>
    {{/if}}
    {{/each}}
</ul>

    4. art-template在分布模板当中的应用与注意事件

      art-template也支持会布模板，这里也使用include这个语法进行导入，但是它与ejs不相同的就是在这里，导入的时候，需要对模板路径加上引号，如下所示

      {{include "../Header.html" obj}}
      1

      在这里，include是导入命令，后面的"./Header.html"代表要导入的分布模板，后面的obj代表的是要传递到子模板当中的数据

      注意：子模板默认在art-template不共享父模板的数据，需要我们手动的向下传递，而ejs默认是共享当前模板的数据

    5. 与EJS模板的异同

      • 在这里，art-template与ejs有很多相同点，art-template支持<%%>语法，也支持{{}}这一种语法，但是在模板里面定义数据的时候，我们必须使用<%var a="abc"%>这种语法格式
      • 在art-template里面，如果要向子模板传递数据，必须先定义好，再传递，不能够说起在include里面去定义
