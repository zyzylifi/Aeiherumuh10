# DOM基础

概念：document object model,文档对象模型，主要的目的就是把网页里面的元素当成对象一样支

网页当中的元素与我们之前学习的对象是非常像的，它有标签名（对象名），它有属性，也有属性值

把网页当中的元素当成对象一样去操作的时候，这样就方便了我们网页的操控

DOM的本质是网页与JS的结合 ，它将我们之前学习的JS应用在了同多页上面，用JS来操作网页

DOM里面又分为两个部分，第一个部分是JS与HTML的结合，第二部分是JS与CSS的结合

  1. 整个页面最终会形成一个document对象
  2. 页面上面所有的元素（标签）最终都会转换成JS里面的对象
  3. 如果是非IE的浏览器，它会根据每ID生成一个对象方便我们直接取用(window.id)

之前我们讲过可以直接通过ID来获取网页当中的标签（这个标签会转化成对象，并且是一个文档对象），如果没有id这个时候怎么办呢？

我们如果想要操作某一个标签元素，最开始应该是怎么样从网页当中获取到这个元素

  1. 通过ID来进行获取页面元素

    document.getElementById("div1");
    1

    通过这各方式去获取，永远只能够获取到0个(null)或1个，如果找到了就是1个，如果找不到就是0个

    注意ID名重复的情况，则找到的永远是第一个

  2. 通过class名称来获取页面元素

    document.getElementsByClassName("div2")';'
    1

    通过这种方式，获取的是一个HTMLCollection的集合（类数组），具备数组的特性，不具备数组的方式，这个集合里面存放的就是我们的DOM对象，如果一个元素都获取不到，它就是一个空的类数组[]

  3. 通过标签名去获取页面元素

    document.getElementsByTagName("p");
document.getElementsByTagName("div")
    1
    2

    通过这种方式，获取到的也是一个HTMLCollection,通过页面上面的某一些标签来获取某一个元素

  4. 通过name属性去获取页面元素

    document.getElementsByName("sex");
    1

    它返回的是一个NodeList的集合（类数组）,具备数组特性，不具备数组方法，这个集合里面存放的就是我们的DOM对象，如果一个元素都获取不到，它就是一个空的类数组[]

上面的4种方式是最基本的4种方式，这种方式没有任何兼容性，它可以解决我们日常工作中的很多问题

    document.querySelector("#div1");  //id="div1"

    这种方式是通过选择器来选择元素,它如果找到了元素就返回1个元素，如果找不到就返回null

  2. 通过querySelectorAll()来选择元素

    document.querySelectorAll("#div1");

    1

    它返回是一个NodeList集合，里面包含了你要查找的元素,找到了就返回，如果找不到就返回一个空的NodeList[]

总结： getElementById和querySelector都是返回一个或0个，如果找到就返回一个，如果找不到就返回空值

querySelectorAll和后面的getElementsBy***返回的是一个集合

    return document.querySelectorAll(selector);
}

# DOM结构

DOM本身是一个对象，我们之前也讲过对象，它具备对象的特点

现在我们来分析一个对象

通过对三种标签的对象，我们最终发现，所以的元素最终都会继承自Node(EventTarget暂时先不做考虑)，我们可以把Node对象做为所有文档对象（页在元素）最终的父级，这个时候，我们只需要把Node对象弄清楚了，那么，所有的元素也都弄清楚了

# Element与Node的区别

  1. 在网页当中，所有的标签它都是一个元素(element)，同时也是一个节点(Node)
  2. 但是并不是所有的节点Node都是元素Element（节点包括元素，文字，注释，回车）

    我在前面插入了文字

    <!-- 这是一个注释 -->

  2. 标签自定义属性，我们又如何去操作呢?

    一般情况下，我们标签是可以随便自定义属性的（只要不与系统的属性重名，均可）,但是，工作当中，W3C推荐我们在自定义的属性前面添加一个data-

    //现在，我们来写一个自定义的属性
<input id="input2" type="text" data-biaogege="标哥哥" data-age="18" />
    1
    2

    如果网页在渲染的时候，发现了一以data-开头的属性，那么它会把当前元素上面所有以data-开头的属性封装一个对象，这个对象的类型是DOMStringMap类型，然后把这个生成好的对象命名成dataset，放在当前的元素上面做为一个属性

    input2.dataset;  //获取所有以data-开头的属性
input2.dataset.biaogege;  //获取具备的属性值 
input2.dataset.age;

# 补充

网页当中有一些特殊的标签会生成一些特殊的属性

  1. document.documentElement指的是html标签生成的对象
  2. document.title指的是网页的标题title标签生成的对象
  3. document.body指的是网页的body标签生成的对象
  4. document.links代表网页link标签所生成的对象的集合，它是一个HTMLCollection
