# 面向对象程序设计（提升篇）

对象与对象的关系主要体现在两个大的方面，一个是继续关系，一个是包含关系

# 对象包含对象

我们先做一个想像，一个班里面包含了多个学生，那么，怎么样构建班级与学生的关系

function Student(userName, sex) {
    this.userName = userName;
    this.sex = sex;
}
//现在来创建班级对象
var classRoom = {
   clasName: "H1904",
   address: "智慧园二楼项目一",
   maxNum: 60,
   classId: 1,
   //这里面有学生
   students: [],
   addStudent:function(s){
       //s就代表你要添加的学生
       if(s instanceof Object&&s.constructor===Student){
           this.students.push(s);
       }
       else{
           console.error("请添加一个学生");
       }
   }
};
var s1=new Student("张三","男");

# 对象继承对象

讲这里，首先就要弄清楚，什么是继承，继承主要表现在哪些方面？

当一个对象继承了另一个对象以后，那么，我们不妨考虑以下几个问题

  1. 如果用户找子级对象拿东西，如果子级对象有，就不找父级对象要了
  2. 如果子级对象没有就找父级对象要

在之前学习方法调用的时候，我们知道方法除了常规范方式调用以外，还可以通过call与aplly来进行，这两种调用方式，可以改变方法内部this的指向

第四步：通过call与apply改变this指向

 /*在这里，我们创建了一个人的构造函数
      这里的构造函数就是提取了公共属性
    */
function Person(userName,sex,age){
    this.userName=userName;
    this.sex=sex;
    this.age=age;
}

//学生的构造函数
function Student(userName,sex,age,sid){
    //子级的应该只保留自己的特点
    this.sid=sid;
    //我们现在已经知道当前Student它是通过构造函数执行的，所以，我们得到this指向当前对象
    //现在，我只需要把当前对象带入到Person里面就可以
    Person.call(this,userName,sex,age);        
}

//老师的构造函数
function Teacher(userName,sex,age,tid){
    //子级的只保留自己的特点
    this.tid=tid;

    Person.call(this,userName,sex,age);
}

var s1=new Student("李昊","男",18,"H1904xxxx");
// var s2=new Student("高自忠","男",22,"H1904xxxx");

var t1=new Teacher("杨标","男",19,"T01");

# 通过prototype实现继承

某一个对象的__proto__指向了它的构造函数的prototype

正是因为有了上面这一种方式 ，所以，我们之前写的__proto__也可以通过构造函数的prototype来实现

function Person(){
    this.sayHello=function(){
        console.log("我在给你打招呼");
    }
}

function Student(sid){
    this.sid=sid;
    //this.__proto__=new Person();  //新方法

Student.prototype=new Person();

# 对象属性的判断是否存在

当我们如果要判断某一个对象是否具备某一个属性的时候，我们需要使用一些特殊的技术

问题：怎么样判断某一个属性是否存在于当前对象之中

function Person(userName,sex,age){
    this.userName=userName;
    this.sex=sex;
    this.age=age;
    this.sayHello=function(){
        console.log("我在向你问好");
    }
}


function Student(userName,sex,age,sid){
    this.sid=sid;
    this.__proto__=new Person(userName,sex,age);
}

var s1=new Student("张三","男",18,"H1904xxxx");
//判断sid这个属性在不在s1里面
//判断userName这个属性在不在s1里面

第一种方式

判断一个对象是否具备某一个属性，我们可以通过关键字in来实现

console.log("sid" in s1);         //true      存在于当前对象
console.log("userName" in s1);    //true      存在于父级对象
console.log("aaa" in s1);         //false     彻底不存在

第二种方式

s1.hasOwnProperty("sid");    //true
s1.hasOwnProperty("age");    //false

通过上面的两个例子，我们发现hasOwnProperty只在当前对象去判断 ，不去父级对象去找

🉐 总结：hasOwnProperty只在当前对象判断，而in关键字会跑到父级对象里面去找。如果存在则返回true,如果不存在则返回false。检测属性是否存在的目的就是为了后期更好的去调用这个属性或属性方式

# 对象属性的覆盖

属性的覆盖在Java或其它编程里面，我们叫重写(override)

按照正常情况当，当我们通过某一个对象调用某一个属性的时候，如果这个对象具备这一个属性，我们就不找它的父级对象去要，如果不具备这个属性就会找父级对象去要，一直找到Object

/*
        用户在这里如果找对象去调某一个属性或方法
        1.如果当前对象有，就不找父级对象要了
        2.如果当前对象没有，就找父级对象要，一直找到Object
 */
function Person(userName){
    this.userName=userName;
}
function Student(userName,sex){
    this.sex=sex;
    this.__proto__=new Person(userName);   //Student继承Person
}

var s1=new Student("张三","男");
console.log(s1.sex);   //sex调用的是当前的属性
console.log(s1.userName);  //调用的是父级的属性
/* userName这个属性 s1自身不存在，所以它要去父级找，然后找到了，就得到了结果，
   这个查找的过程是系统自己去完成
*/

案例 ：假如你爸有辆车，你如果没有车，你可以使用你爸的车，但是如果你自己也有了一辆，那么，你就不会找你爸要车了

function Father(){
    this.car="五菱宏光";
}

function Son(){
    //先实现继承关系 
    this.__proto__=new Father();
    this.driver=function(){
        //如果你有car，就不找父级，如果没有就找父级
        console.log("我在开车，我的车是"+this.car);
    }
}

var s=new Son(); //现在创建了一个儿子

现在在刚刚的代码上面，我们已经创建了两个构造函数，并且让Son的继承了Father

当s儿子在调用driver这个方法的时候，它使用的this.car指向的是父亲的“五菱宏光”

突然有一天，儿子有钱了，自己买了一辆宝马车，这个时候，儿子开车就不在需要父亲的车了，这个时候，怎么处理呢

function Father(){
    this.car="五菱宏光";
}

function Son(){
    //先实现继承关系 
    this.__proto__=new Father();
    this.driver=function(){
        //如果你有car，就不找父级，如果没有就找父级
        console.log("我在开车，我的车是"+this.car);
    }
    this.car="宝马";  
}

var s=new Son(); //现在创建了一个儿子
s.driver();    //儿子在开宝马车
